# Adding a New Article

## Quick Steps

1. Create `src/content/YourArticle.md` with YAML frontmatter (see below)
2. Write the body in GitHub-flavored markdown (headings, tables, footnotes, code fences all supported)
3. Link internally with `[text](/lens/{key})` or `[text](/articles/{slug})` — see the link behavior section
4. Run `npm run typecheck && npm run format:check && npm run test` — then `npm run build` to confirm the route was discovered
5. Done — `import.meta.glob` auto-registers every `src/content/*.md` file; no imports or catalog edits required

## Frontmatter Reference

```yaml
---
slug: my-article                    # required — URL-safe, becomes /articles/my-article
title: My Article Title             # required — H1 and browser title
summary: A short description.       # recommended — shown on archive cards and in SEO
tag: guide                          # optional — "guide" | "article" | "announcement"
series: focusing-architectures      # optional — kebab-case id shared by series members
seriesOrder: 3                      # optional — integer; 0 = series landing page
toc: true                           # optional — shows the hovering table of contents
---
```

| Field | Required | Purpose |
|---|---|---|
| `slug` | ✅ | URL path segment. Must be unique across all articles. Missing/duplicate slugs cause the article to be silently skipped by the metadata generator. |
| `title` | ✅ | Display title used in breadcrumbs, cards, SEO, and JSON-LD. |
| `summary` | recommended | One or two sentences shown on the `/articles` archive, homepage card, and `<meta name="description">`. |
| `tag` | optional | Visual badge category on cards. |
| `series` | optional | Groups related articles. Members collapse to one card on the archive and one slot on the homepage. |
| `seriesOrder` | optional | Sort order within the series. `0` marks the landing/index page; primers are `1, 2, 3…` |
| `toc` | optional | When `true`, renders `ArticleTOC` — the floating right-side outline with scrollspy. Recommended for long pieces with ≥3 H2 sections. |

> **Dates are automatic.** `publishedOn` and `lastModified` are derived from git history at build time. Do not add them to frontmatter.

## Series

A **series** is a cluster of related articles identified by a shared `series` value. The member with `seriesOrder: 0` is the **landing page** — the canonical entry point.

What the pipeline does for you automatically:

- **`/articles` archive** — renders the series as a single `SeriesCard` listing every member, instead of one card per article.
- **Homepage "Recent articles"** — shows the landing page only; primers don't consume homepage slots.
- **Breadcrumb on each member** — reads *Home / Articles / {Series landing title} / {Current title}*.
- **In-page back link** — points at the series landing page rather than `/articles`.
- **JSON-LD breadcrumbs** mirror the visible path.

### Minimum to create a series

1. A landing page with `seriesOrder: 0` that links to every member.
2. One or more member articles sharing the same `series` id, with `seriesOrder` ≥ 1.

The landing page is just another markdown file — no special component, no routing.

## Math / LaTeX

Math is rendered via KaTeX (through `remark-math` + `rehype-katex`). Both article pages and lens analysis (`DescriptionPanel`) support it — no per-article opt-in.

- **Inline math:** wrap in single dollar signs — `$S_I = -A^2 h\,\Delta(u/n)$`
- **Display math:** wrap in double dollar signs on their own line —

  ```
  $$\Delta S_I \approx 4\delta S_{II}$$
  ```

- **Supported syntax:** the [KaTeX function list](https://katex.org/docs/supported.html). Common macros (`\frac`, `\tfrac`, `\bar`, `\mathcal`, `\sum`, `\cdots`, `\approx`, `\propto`, Greek letters) all work.
- **Unsupported:** `\begin{align}`/`\end{align}` environments, `\label`/`\ref`, `\cite`. Use display-math blocks and prose references instead.
- **Escaping a literal dollar sign** in prose (e.g., USD amounts): write `\$` so `remark-math` doesn't interpret it as math.

Global KaTeX CSS is imported once in `src/main.tsx`; the fonts are emitted as hashed assets during build.

## Hovering Table of Contents

Opt in via `toc: true`. Behavior:

- Extracts H2/H3 headings (skips code fences).
- On viewports ≥ 1200px: docks to the right gutter, sticky, with scrollspy highlighting the active section.
- Below 1200px: collapses to a floating "Contents" toggle button in the bottom-right.
- Auto-hides when fewer than 3 headings are found (tunable via `minHeadings` prop if mounting the component directly).

Anchor IDs are generated by `rehype-slug` during markdown rendering and by the TOC's extraction function (both use `github-slugger`, so they stay in sync).

## Link Behavior

`ArticlePage`'s custom `<a>` renderer routes links based on their href:

| Href starts with | Renders as | Behavior |
|---|---|---|
| `/lens/…` | `<a target="_blank">` | Opens in a new tab — keeps the article in view while the reader explores the interactive viewer |
| `/…` (any other internal) | React Router `<Link>` | SPA navigation, same tab |
| `#…` | `<a href="#…">` | In-page anchor with `scrollMarginTop: 88` to clear the sticky nav bar |
| `http…` | `<a rel="noopener noreferrer">` | External, same tab |

The anchor handler also forwards the `id` prop so `remark-gfm` footnote references (`user-content-fnref-N`) and back-references resolve in both directions.

## Key Files

- **Markdown location:** `src/content/*.md` (auto-discovered)
- **Frontmatter parsing:** `scripts/generate-build-metadata.mjs` — extend here if you add a new frontmatter field
- **Registry + series grouping:** `src/utils/homepageContent.ts` — `ARTICLES`, `HOMEPAGE_ARTICLES`, `ARTICLE_SERIES`, `ARTICLE_CONTENT`
- **Page rendering:** `src/pages/ArticlePage.tsx` (single article), `src/pages/ArticlesPage.tsx` (archive)
- **TOC component:** `src/components/display/ArticleTOC.tsx`
- **Series card:** `src/components/homepage/SeriesCard.tsx`
- **Generated metadata:** `src/generated/build-metadata.json` (gitignored; regenerated on every build)

## Verification

After creating or editing an article:

```bash
npm run typecheck && npm run format:check && npm run lint && npm run test
npm run build
```

Then confirm:

- `src/generated/build-metadata.json` lists the new slug under `articles[]` and `/articles/{slug}` under `routes[]`
- `dist/articles/{slug}/index.html` exists (prerendered)
- `dist/sitemap.xml` includes `https://opticalbench.net/articles/{slug}` at priority 0.6
- For TOC-enabled articles, grep the prerendered HTML for `id="…"` on H2/H3 elements to confirm `rehype-slug` ran
